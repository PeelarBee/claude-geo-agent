# LLM Visibility Prompt Library — Index

40 prompts across 8 modules. Generated by `claude-geo-agent` as `06-LLM-PROMPTS-TO-RUN.md`, fully instanced for each business.

## Not a Measurement Report

This index defines prompt groups and recommended run modes. It does not contain measured results, visibility scores, model responses, or client-facing claims.

A measurement exists only when a prompt was executed in a fresh/cold context through a configured provider or documented manual run, with provider, model if known, prompt text, raw response or response summary, citations/sources, extraction result, and run date/time logged.

**Default light starter run:** 10, 11, 13, 20, 23, 25, 30, 90

**Fuller client-facing baseline run:** 10, 11, 13, 20, 21, 22, 23, 24, 25, 30, 41, 50, 90

---

## Cold-Context Rule

Measurement prompts must be run in a fresh model context with no prior brand briefing, no audit notes, and no previous answers. Otherwise, results may be contaminated by context.

- Do not run prompts inside the same conversation used to create the audit, CONFIG, or recommendations.
- Do not include brand briefings, site summaries, previous audit findings, or hidden context unless the specific test is intentionally designed as a warm-context test and labeled as such.
- Warm-context tests must not be mixed with cold-context baseline results.

---

## Provider Boundary

`SERPER_API_KEY` is search only. It can support Search Evidence, competitor discovery, external authority checks, brand mentions in search surfaces, and source discovery. It cannot measure ChatGPT, Claude, Gemini, Groq, OpenAI, Perplexity, or Bing Copilot answer visibility.

Measured LLM visibility requires at least one configured LLM provider:

- `GEMINI_API_KEY` or `GOOGLE_API_KEY` for Gemini prompt execution
- `GROQ_API_KEY` for Groq model prompt execution
- `OPENAI_API_KEY` for OpenAI prompt execution
- `ANTHROPIC_API_KEY` for Claude prompt execution
- `PERPLEXITY_API_KEY` for Perplexity prompt execution

Do not claim ChatGPT UI results if only OpenAI API was used. Say `OpenAI API` or `OpenAI model`. Mark Bing Copilot as `Manual / Not run` unless an actual supported measurement path exists.

- `SERPER_API_KEY` results must be labeled `Search Evidence`.
- Serper/search results can support competitor discovery and source discovery, but cannot populate measured LLM visibility fields.
- If no LLM provider is configured, `07-LLM-VISIBILITY-RESULTS.md` must say exactly: `Status: Not run — no LLM provider configured`
- If prompts were generated but not executed, use: `Status: Manual run required — prompt library generated only`

## Required Logging Fields

Every executed prompt run must log:

- Provider
- Provider interface: API / Manual UI / Other
- Model, if known
- Prompt ID
- Prompt group
- Prompt text
- Date/time of run
- Fresh/cold-context status
- Raw response or response summary
- Brand mentioned: yes/no
- Domain cited: yes/no
- Competitors mentioned
- Citations/sources, if any
- Sentiment/framing
- Extraction result
- Evidence Status
- Evidence Source
- Evidence Source Type
- Confidence
- Limitations / notes

## Evidence Status Values

Approved Evidence Status values:

Observed / Measured / Search Evidence / Inferred / Not run / Not available / Unknown

- Executed LLM provider responses may be `Measured`.
- Search API results must be `Search Evidence`.
- Interpretation prompts based on prior results are `Inferred` unless they directly summarize measured run logs.
- Generated prompts that were not executed are `Not run`.
- Missing provider data is `Not available` or `Not configured` depending on the output table.

## Run Modes

The prompt library is organized as a phased system. Different objectives run different prompt groups.

| Mode | Purpose | Prompt groups |
|---|---|---|
| `quick-check` | Fast diagnostic; does not produce measured LLM visibility | Pre-flight checks only |
| `monitor` | Tracks prompt-based visibility only if configured providers are actually run | Measurement `10-17` + Extraction `20-25` + Validation `90` |
| `full-audit` | Complete GEO audit and action plan; measurement-dependent sections are marked `Not run` when providers are missing | Discovery `01-05` + Measurement `10-17` + Extraction `20-25` + Validation `90` + Interpretation `30-34` + Audit `40-48` + Action `50-53` + Learning `60-61` |
| `refresh` | Updates an existing audit from new evidence only; stale prior results must keep their original run date | Interpretation `30-34` + selected Audit `40-48` + Action `50-53` + Learning `60-61` |

Use the full library for a complete engagement. Use a narrower mode when the task is only monitoring, refreshing, or checking technical readiness.

*Run after every measurement prompt. Parses LLM responses into structured JSON data. Never assume clean JSON from the model — these prompts handle defensive parsing.*

- Extraction prompts must only parse actual executed LLM responses.
- Extraction prompts must not fabricate JSON fields from missing responses.
- If raw response is missing, extraction status must be `Not run` or `Not available`.
- Extraction output should preserve uncertainty rather than infer mentions/citations.

## Running the Prompts

### Option A — Manually in any LLM interface
Copy individual prompts from `06-LLM-PROMPTS-TO-RUN.md` and paste into ChatGPT, Claude, Perplexity, or Gemini. Log results manually.

Manual UI runs must be clearly labeled as `Manual UI`. Capture screenshots or copied raw responses when practical. Do not mix manual UI results with API results without labeling the provider interface.

For every manual run, log:

- Provider
- Provider interface: API / Manual UI / Other
- Model, if known
- Date/time
- Prompt text
- Raw response
- Citations/sources, if any
- Whether the run was fresh/cold-context
- Extraction status

### Option B — Workflow automation (recommended for ongoing monitoring)
Use a workflow tool to run Module 2 prompts across configured providers on a schedule, when the relevant integrations are available and verified:

- **[Relay App](https://relay.app)** — free, simple, purpose-built for running prompts across LLMs in parallel
- **[n8n](https://n8n.io)** — self-hosted workflow automation, more control
- **[Make](https://make.com)** — no-code automation with LLM integrations

Depending on available integrations and configuration, workflow automation may support:
1. Running all 8 measurement prompt groups across configured LLM providers monthly
2. Feeding responses through the Module 3 extraction prompts automatically
3. Producing a compiled report only when that workflow is actually configured

# LLM Visibility Prompt Library — Index

40 prompts across 8 modules. Generated by `claude-geo-agent` as `06-LLM-PROMPTS-TO-RUN.md`, fully instanced for each business.

**Important:** This index describes the prompt library. It is not a measurement report. LLM visibility is only measured after prompts are executed in fresh model contexts through configured LLM providers.

**Default light starter run:** 10, 11, 13, 20, 23, 25, 30, 90

**Fuller client-facing baseline run:** 10, 11, 13, 20, 21, 22, 23, 24, 25, 30, 41, 50, 90

---

## Cold-Context Rule

Measurement prompts must be run in a fresh model context with no prior brand briefing, no audit notes, and no previous answers. Otherwise, results may be contaminated by context.

---

## Provider Boundary

`SERPER_API_KEY` is search only. It can support search evidence, competitor discovery, external authority checks, brand mentions in search surfaces, and source discovery. It cannot measure ChatGPT, Claude, Gemini, Groq, OpenAI, Perplexity, or Bing Copilot answer visibility.

Measured LLM visibility requires at least one configured LLM provider:

- `GEMINI_API_KEY` or `GOOGLE_API_KEY` for Gemini prompt execution
- `GROQ_API_KEY` for Groq model prompt execution
- `OPENAI_API_KEY` for OpenAI prompt execution
- `ANTHROPIC_API_KEY` for Claude prompt execution
- `PERPLEXITY_API_KEY` for Perplexity prompt execution

Do not claim ChatGPT UI results if only OpenAI API was used. Say `OpenAI API` or `OpenAI model`. Mark Bing Copilot as `Manual / Not run` unless an actual supported measurement path exists.

## Run Modes

The prompt library is organized as a phased system. Different objectives run different prompt groups.

| Mode | Purpose | Prompt groups |
|---|---|---|
| `quick-check` | Fast diagnostic, no deep LLM measurement | Pre-flight checks only |
| `monitor` | Ongoing LLM visibility tracking | Measurement `10-17` + Extraction `20-25` + Validation `90` |
| `full-audit` | Complete GEO audit and action plan | Discovery `01-05` + Measurement `10-17` + Extraction `20-25` + Validation `90` + Interpretation `30-34` + Audit `40-48` + Action `50-53` + Learning `60-61` |
| `refresh` | Update an existing audit from new data | Interpretation `30-34` + selected Audit `40-48` + Action `50-53` + Learning `60-61` |

Use the full library for a complete engagement. Use a narrower mode when the task is only monitoring, refreshing, or checking technical readiness.

## Running the Prompts

### Option A — Manually in any LLM interface
Copy individual prompts from `06-LLM-PROMPTS-TO-RUN.md` and paste into ChatGPT, Claude, Perplexity, or Gemini. Log results manually.

For every manual run, log:

- Provider
- Model, if known
- Date/time
- Prompt text
- Raw response
- Citations/sources, if any
- Whether the run was fresh/cold-context
- Extraction status

### Option B — Workflow automation (recommended for ongoing monitoring)
Use a workflow tool to run all Module 2 prompts in parallel across multiple LLMs on a schedule:

- **[Relay App](https://relay.app)** — free, simple, purpose-built for running prompts across LLMs in parallel
- **[n8n](https://n8n.io)** — self-hosted workflow automation, more control
- **[Make](https://make.com)** — no-code automation with LLM integrations

With any of these tools you can:
1. Run all 8 measurement prompt groups across configured LLM providers monthly
2. Feed responses through the Module 3 extraction prompts automatically
3. Receive a compiled report by email
